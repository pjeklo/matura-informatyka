ŚCIĄGA Z MICROSOFT ACCESS

1. TWORZENIE I ZARZĄDZANIE BAZAMI DANYCH

Tworzenie nowej bazy danych

  1. Plik → Nowy → Pusta baza danych
  2. Podaj nazwę i lokalizację pliku
  3. Kliknij "Utwórz"

Tworzenie tabeli

  1. Zakładka "Tworzenie" → Tabela
  2. Definiowanie pól:
    • Nazwa pola (np. ID_Ucznia, Nazwisko)
    • Typ danych (Tekst, Liczba, Data/Godzina, Autonumerowanie)
    • Opis (opcjonalnie)
  3. Ustawienie klucza podstawowego:
    • Zaznacz pole → Narzędzia tabel → Projektowanie → Klucz podstawowy

Typy danych

  • Tekst - dla krótkich ciągów znaków (do 255 znaków)
  • Memo - dla długich ciągów znaków
  • Liczba - dla wartości liczbowych
  • Data/Godzina - dla dat i czasów
  • Waluta - dla wartości pieniężnych
  • Autonumerowanie - automatycznie przypisywane numery (dobry wybór na klucz podstawowy)
  • Tak/Nie - dla wartości logicznych
  • Obiekt OLE - dla grafik, plików
  • Hiperłącze - dla adresów URL
  • Załącznik - dla załączonych plików

2. IMPORTOWANIE DANYCH

Import z pliku tekstowego/CSV

  1. Zakładka "Zewnętrzne dane" → Tekst
  2. Wybierz plik → Importuj
  3. Wybierz typ pliku (rozdzielany/o stałej szerokości)
  4. Wybierz separator (przecinek, średnik, tabulator)
  5. Określ typy danych dla kolumn
  6. Wybierz istniejącą tabelę lub utwórz nową
  7. Opcjonalnie: ustaw klucz podstawowy

Po imporcie

  1. Sprawdź poprawność danych
  2. Dodaj klucz podstawowy, jeśli to konieczne:
    • Otwórz tabelę w widoku projektu
    • Zaznacz odpowiednie pole
    • Kliknij ikonę "Klucz podstawowy"

3. RELACJE MIĘDZY TABELAMI

Tworzenie relacji

  1. Narzędzia bazy danych → Relacje
  2. Dodaj tabele do widoku relacji (kliknij prawym → Pokaż tabelę)
  3. Przeciągnij pole z klucza podstawowego jednej tabeli do pola klucza obcego w drugiej tabeli
  4. W oknie dialogowym "Edytuj relacje":
    • Zaznacz "Wymuszaj więzy integralności"
    • Opcjonalnie: "Kaskadowe aktualizowanie..." i "Kaskadowe usuwanie..."
  5. Kliknij "Utwórz"

Typy relacji

  • Jeden-do-jednego (1:1) - każdy rekord w tabeli A jest powiązany z dokładnie jednym rekordem w tabeli B
  • Jeden-do-wielu (1:N) - każdy rekord w tabeli A może być powiązany z wieloma rekordami w tabeli B
  • Wiele-do-wielu (M:N) - realizowane przez tabelę łącznikową

4. KWERENDY (ZAPYTANIA)

Tworzenie kwerendy wybierającej

  1. Zakładka "Tworzenie" → Kreator kwerend / Projekt kwerendy
  2. Dodaj potrzebne tabele
  3. Dodaj pola do siatki projektu (kliknij dwukrotnie na nazwę pola)
  4. Określ kryteria wyboru w wierszu "Kryteria"
  5. Ustaw sortowanie
  6. Uruchom kwerendę (przycisk "Uruchom")

Podstawowe operatory w kryteriach

  • =, <, >, <=, >=, <>
  • BETWEEN wartość1 AND wartość2
  • IN (wartość1, wartość2, ...)
  • LIKE "wzorzec" (gdzie % zastępuje dowolny ciąg znaków, _ zastępuje pojedynczy znak)
  • IS NULL, IS NOT NULL
  • Operatory logiczne: AND, OR, NOT

Kwerendy z funkcjami agregującymi

  1. Otwórz kwerendę w widoku projektu
  2. Kliknij przycisk "Sumy" na pasku narzędzi
  3. W wierszu "Funkcja" wybierz odpowiednią funkcję dla pola:
    • Suma - sumuje wartości
    • Średnia - oblicza średnią
    • Min - znajduje wartość minimalną
    • Max - znajduje wartość maksymalną
    • Policz - zlicza wartości

Przykładowe rozwiązanie problemu "Nie zdefiniowana funkcja Maksimum w wyrażeniu"

  1. Użyj angielskiej nazwy funkcji: Max zamiast Maksimum
  2. Sprawdź ustawienia regionalne systemu
  3. Upewnij się, że format daty jest poprawny (w zależności od ustawień regionalnych)

5. JĘZYK SQL

Podstawowa struktura zapytania SELECT

SELECT pole1, pole2, pole3
FROM tabela1
WHERE warunek
ORDER BY pole1 [ASC|DESC];

Łączenie tabel (JOIN)

SELECT t1.pole1, t2.pole2
FROM tabela1 AS t1
INNER JOIN tabela2 AS t2
ON t1.klucz = t2.klucz_obcy
WHERE warunek;

Typy złączeń (JOIN)

  • INNER JOIN - tylko rekordy z dopasowaniem w obu tabelach
  • LEFT JOIN - wszystkie rekordy z lewej tabeli i dopasowane z prawej
  • RIGHT JOIN - wszystkie rekordy z prawej tabeli i dopasowane z lewej

Funkcje agregujące w SQL

SELECT pole1, COUNT(*) AS liczba, AVG(pole2) AS srednia
FROM tabela
GROUP BY pole1
HAVING COUNT(*) > 1
ORDER BY COUNT(*) DESC;

Kwerenda tworząca nową tabelę

SELECT pole1, pole2 INTO NowaTabelaNazwa
FROM IstnieTabelaNazwa
WHERE warunek;

Kwerenda aktualizująca

UPDATE TabelaNazwa
SET pole1 = wartość1, pole2 = wartość2
WHERE warunek;

Kwerenda usuwająca

DELETE FROM TabelaNazwa
WHERE warunek;

CZĘSTO UŻYWANE FUNKCJE W ACCESSIE

Tekstowe

  • Left(tekst, liczba_znaków) - zwraca określoną liczbę znaków od lewej
  • Right(tekst, liczba_znaków) - zwraca określoną liczbę znaków od prawej
  • Mid(tekst, start, liczba_znaków) - zwraca znaki ze środka tekstu
  • Len(tekst) - zwraca długość tekstu
  • Trim(tekst) - usuwa spacje z początku i końca

Datowe

  • Date() - bieżąca data
  • Year(data) - rok z daty
  • Month(data) - miesiąc z daty
  • Day(data) - dzień z daty
  • DateDiff("interwał", data1, data2) - różnica między datami

Liczbowe

  • Round(liczba, miejsca_dziesiętne) - zaokrągla liczbę
  • Int(liczba) - zwraca część całkowitą liczby
  • Abs(liczba) - wartość bezwzględna

Logiczne

  • IIf(warunek, wartość_jeśli_prawda, wartość_jeśli_fałsz) - warunkowe zwracanie wartości
  • IsNull(wyrażenie) - sprawdza, czy wyrażenie jest NULL
  • Nz(wyrażenie, wartość_jeśli_null) - zastępuje NULL określoną wartością
